Purpose

Checking out how git tracks the changes I make in another program.

Explanation

By changing the branch via the terminal the program running my current project automatically is switched to the version that resides on a new git branch.

Your copy on your computer hardrive is separate from the versions that are stored within your local and global git repositories.

Think of it like having two separate bookshelevs with the same books but one can be thrown out but you would still have the copies of the same books elsewhere.

Process

Step 1: Open Terminal

Step 2: Create a branch off of the master branch.

$ git branch [branch name]

Step 3: Checkout the new branch.

$ git checkout [branch name]

Step 4: Add and commit your first change.

$ git commit -a -m "Commiting my first change"
